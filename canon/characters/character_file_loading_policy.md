# Character File Loading Policy

## Core character files

Для персонажа в текущей сцене по умолчанию загружаются только основные файлы:

- `characters/<id>/main.yaml` — короткая идентичность, роль, голос, внешний вид, активный характер.
- `characters/<id>/character.yaml` — поведение, границы, стиль реплик, привычки, запреты образа.
- `characters/<id>/knowledge.yaml` — что персонаж знает, чего не знает, что скрывает и что может раскрывать.

## Conditional past file

`characters/<id>/past.yaml` не является файлом постоянной загрузки.

Его можно добавлять только если текущая сцена прямо касается прошлого, памяти, Самуэля, связи Акиры и Райдена, ребёнка, кольца, шрама, лаборатории, Эхо, кайросов, пространства между, срыва или самоблокировки.

## Archive/legacy files

Не использовать как активные источники:

- `*_main_profile.yaml`
- `*_hidden_past.yaml`
- `*_knowledge_connections.yaml`

Если они остались физически, это архив/мусор для ручного удаления, а не current context.

## Rule

Сцена не должна тянуть полный скрытый пакет персонажа только потому, что персонаж стоит рядом.

Знать персонажа в сцене = загрузить core.
Копать прошлое персонажа = загрузить `past.yaml` только по явному смысловому триггеру.
