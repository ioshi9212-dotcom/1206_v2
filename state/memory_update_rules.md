# Memory update rules — 1206 v2

После сцены сохранять только значимые изменения:

- кто что узнал;
- кто что перестал не знать;
- кто что начал скрывать;
- кто что ошибочно понял;
- отношения;
- репутацию;
- слухи;
- предметы;
- смену локации/фазы дня;
- продвижение календаря;
- сюжетные обязательства.

Backend не выводит изменения из прозы автоматически: изменения нужно отправлять через `apply-turn-result`.

## Правило знаний v2 на русском

`state/knowledge_state.json` хранит знания простыми русскими пунктами.

Карточки персонажей хранят неизменное: внешность, характер, голос, энергию, устойчивые мотивы, авторскую правду/hidden past.

`state/knowledge_state.json` хранит изменяемое: персонаж знает, персонаж не знает, персонаж скрывает от другого, персонаж ошибочно считает, история знаний.

## Как обновлять знание

Если персонаж узнал факт, нельзя просто добавить новый пункт сверху и оставить старый конфликтующий пункт в `не знает`.

Нужно сделать переход:

- убрать конфликтующий пункт из `не знает`;
- добавить актуальный пункт в `знает`;
- записать событие в `история знаний`;
- сохранить источник и дату, если они известны.

## Каждые 15 значимых ходов

Делать компактное обновление `knowledge_state`: убрать дубли, убрать устаревшие пункты из `не знает`, сохранить свежие раскрытия в `история знаний`, оставить только актуальные знания и незнания.
