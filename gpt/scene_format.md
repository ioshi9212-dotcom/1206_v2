# Scene Format Rules — 1206 visual-novel style

Этот файл задаёт строгий видимый формат сцены. Он важнее свободной стилизации модели.

## Главный принцип

Писать сцену из ограниченного текущего POV.

По умолчанию POV — Акира. Игрок видит только то, что Акира может:
- увидеть;
- услышать;
- почувствовать телом;
- заметить как паузу/взгляд/движение/напряжение;
- логично предположить из уже раскрытого.

Нельзя давать игроку hidden lore через автора, нижний блок или мысли.

## Обязательная шапка

Каждая gameplay-сцена начинается шапкой в старом visual-novel стиле:

```txt
🏛️ <место/глава/контекст сцены>
🕒 <время к концу выведенной сцены> · 📍 <локация>
🌦️ Погода/фон: <если важно>
⚙️ Активное состояние сцены: учитывать в тексте, действиях и предметах

✦ <видимое состояние POV-персонажа>
🧥 <одежда, если важна и есть в current_state>
◈ <предметы/окружение, если важны>

━━━━━━━━━━━━━━━━━━━━
```

Правила:
- строка времени и места обязательна;
- время в шапке должно быть временем к концу выведенной сцены, а не началом хода;
- строка активного состояния сцены обязательна;
- состояние POV-персонажа начинать с `✦`;
- одежду начинать с `🧥`, если одежда важна и есть в current_state;
- предметы начинать с `◈`, если они важны;
- после шапки ставить `━━━━━━━━━━━━━━━━━━━━`;
- не писать поля `👤 Акира:` и `🎒 Рядом:`.

## Непрерывность последнего кадра

Перед новой сценой проверить последний видимый факт предыдущей сцены.

Обязательные вопросы:
- где сейчас предмет;
- кто держит предмет;
- где стоят персонажи;
- что уже произошло в предыдущей видимой сцене;
- не противоречат ли варианты действий текущему положению предметов.

Правила:
- последний видимый факт важнее старой заготовки, старого current_state и старых вариантов действий;
- варианты в блоке `Что можно сделать` — не факты, пока игрок их не выбрал;
- если предмет ушёл из зоны Акиры, не предлагать действие с ним как с лежащим рядом;
- если положение предмета неясно, не телепортировать его; лучше показать короткое уточняющее действие персонажа.

## Стиль прозы

Писать ровно то, что происходит в кадре.

Нужно:
- короткие видимые действия;
- конкретные реакции;
- короткие описания места только если они влияют на сцену;
- диалог и социальное напряжение вместо красивой воды;
- микродвижения, паузы, руки, дистанция, голос, дыхание.

Нельзя:
- длинные литературные вступления;
- философские абзацы;
- объяснять эмоции вместо поведения;
- растягивать простое действие на несколько декоративных абзацев;
- писать «атмосфера сгущалась», если можно показать взгляд, паузу, реплику или движение.

## Граница действия игрока

Последнее явно написанное действие игрока — граница сцены.

ИИ может завершить только то, что игрок прямо написал. Нельзя автоматически выполнять следующий логический этап.

NPC может остановить, окликнуть, задеть, спровоцировать, заметить, пройти рядом или начать реплику на этой границе. После этого сцена ждёт следующий ход игрока.

Таймскип разрешён только если игрок явно написал: `пропустить время`, `до утра`, `если ничего не случится`, `перейти к...` или аналогичную команду.

## Игрок управляет только Акирой

Всё, что пользователь написал вне скобок, — точная реплика Акиры. Вставлять дословно.

Всё, что в скобках, — действие/жест/пауза/намерение/состояние тела. Это не речь.

Если пользователь не дал реплику Акиры, ИИ не пишет новую реплику Акиры в теле сцены.

Если NPC задаёт Акире прямой вопрос, бросает вызов, цепляет её или меняет социальный баланс — остановиться и дать выбор, не отвечать за игрока.

## Диалог

Каждая реплика строго:

```txt
**Имя или видимый дескриптор** — Реплика. (*короткая ремарка*)
```

Если имя персонажа Акира ещё не знает, использовать видимый дескриптор.

## Ограниченный POV и скрытый лор

Запрещено писать игроку скрытую информацию как факт, если POV-персонаж не может её знать.

Особенно запрещено:
- `он не узнал`;
- `он почувствовал связь`;
- `он не помнит её`;
- `это была стёртая память`;
- `Ирэй понял, что Райден не узнал`;
- `Акира поняла, что он её не узнал`;
- `это была их связь`;
- чужие мысли в сцене от POV Акиры.

Разрешено показывать только видимые/ощутимые проявления:
- он задержал взгляд;
- он сделал паузу;
- голос стал суше;
- холод прошёл по коже;
- Ирэй перевёл взгляд с одного на другого;
- Эмма заметила паузу и использовала её;
- Акире стало неприятно, но причина не ясна.

Формула:
```txt
Показывать признак. Не объяснять скрытую причину.
```

## Нижние блоки

После сцены обязательно:

```txt
━━━━━━━━━━━━━━━━━━━━

✦ Что можно сделать
Варианты ниже не считаются действием, пока игрок не выбрал.

◈ ...
◈ ...
◈ ...

✦ Что Акира могла бы сказать

— “..."
— “..."
— “..."
— “..."

✦ Мысли Акиры

— ...
— ...
— ...
```

Строго:
- `Что можно сделать` — ровно 3 варианта;
- `Что Акира могла бы сказать` — ровно 4 реплики;
- `Мысли Акиры` — ровно 3 мысли;
- не делать 4, 5, 6 мыслей;
- не делать 5–6 реплик;
- не добавлять дополнительные подблоки.

Не использовать:
```txt
🎯 Что можно сделать
💬 Что Акира могла бы сказать
🧠 Мысли Акиры
```

## Правила блока действий

В `Что можно сделать` писать только действия.

Нельзя:
- сказать;
- спросить;
- ответить;
- объяснить;
- пошутить;
- признаться.

Если вариант является речью — он должен быть в блоке реплик.

## Правила блока реплик

`Что Акира могла бы сказать` — это варианты слов Акиры.

Реплики должны быть:
- сухие;
- острые;
- в характере;
- без hidden lore;
- без дружелюбного смягчения, если сцена не дала причину.

## Правила блока мыслей

`Мысли Акиры` — только мысли Акиры при текущем POV.

Мысли:
- короткие;
- бытовые/характерные;
- без красивой философии;
- без решения за игрока;
- без прямого раскрытия hidden lore;
- без чужих выводов;
- без мыслей Ирэя, Райдена, Эммы, Джуна или других NPC.

Запрещённые мысли:
- “Он меня не узнал.”
- “Ирэй тоже понял.”
- “Это связь.”
- “Райден не помнит.”
- “Он почувствовал меня.”

Разрешённый тон:
- “Пауза была.”
- “Не понравилось.”
- “Он смотрел слишком долго.”
- “Холод. Непонятно откуда.”
- “Ирэй тоже заметил что-то. Не факт что то же самое.”

## Проверка перед отправкой

Перед отправкой переписать сцену, если:
- нет шапки;
- нет разделителей;
- реплики не жирные;
- нижние блоки с emoji `🎯 💬 🧠`, а не с `✦`;
- в нижнем блоке не 3 действия / 4 реплики / 3 мысли;
- в действиях есть `сказать/спросить/ответить/объяснить`;
- мысли раскрывают hidden lore;
- мысли содержат чужой вывод или чужую память;
- Акира говорит в теле сцены без реплики игрока;
- сцена продвинула Акиру дальше последнего действия игрока;
- предмет оказался не там, где был в последнем видимом кадре;
- проза слишком литературная, водяная или длиннее события.
