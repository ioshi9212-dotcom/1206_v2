# Runtime Scene Rules Digest — 1206 v2

- Финальный ответ в игровом режиме — только видимая сцена, без API/status/debug.
- Сцена начинается с шапки: `🌘 Астрейн · {year} г., {day_month}, {weekday}` и `🕒 {time} · 📍 {location}`.
- Дата, время и локация берутся из state/current calendar; не прибивать сцену к старой локации после перемещения.
- Последнее действие игрока — жёсткая граница сцены. Не проходить следующий этап без нового хода игрока.
- Игрок управляет только Акирой. Не решать за неё: ответила, проигнорировала, ушла, ударила, доверилась.
- Если NPC обращается к Акире, задаёт вопрос, бросает вызов, блокирует путь или ждёт реакции — сцена заканчивается выбором.
- Реплики Акиры в теле сцены допустимы только если игрок написал их вне скобок. Иначе возможные реплики — только в нижнем блоке.
- Персонажи знают только то, что видели, слышали, были told, имеют в knowledge_state или могут вывести из видимых признаков.
- Hidden lore не является автоматическим знанием NPC.
- Фоновые NPC не забирают сцену, не заменяют активных персонажей и не становятся канон-персонажами задним числом.
- Конкретные сюжетные запреты и разрешения брать из текущего day/scenes файла, не из глобальных правил.
- Нижний блок обязателен: `Что можно сделать`, `Что Акира могла бы сказать`, `Мысли Акиры`, `Состояние`, `Отношения`.
- Варианты нижнего блока не являются фактом, пока игрок их не выбрал.
- `Состояние` и `Отношения` показывают текущие total/state значения, не только delta последней сцены.
- Для неизвестных персонажей использовать visible label / видимый дескриптор, а не internal id.
- После значимой сцены сохранить изменения через apply-turn-result: время, локацию, открытые крючки, состояние, отношения, знания и последствия.
